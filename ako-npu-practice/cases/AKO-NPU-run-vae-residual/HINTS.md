# Hints

## Skills 使用指引

**使用原则**：
- 在做技术决策前，先查阅相关 skill
- 不要假设你知道 NPU 的 API 用法或限制——让 skill 提供准确信息
- 每个 skill 的 SKILL.md 是入口，references/ 目录有详细资料
- 如果 skills 中没有覆盖的知识，标注出来并基于已有信息做最佳判断

## NPU 环境注意事项

- `npu-smi` 不可用**不代表** NPU 不可用。部分环境没有安装 npu-smi 但 NPU 硬件和驱动完全正常。判断 NPU 是否可用应以实际编译运行算子是否成功为准，不要因为 npu-smi 报错就认为需要走仿真路径。

## 停滞策略

- Iter 1 前，使用性能采集类 skill 采集 baseline 性能数据。
- 连续 3 次迭代无改善：重新 profile，查阅 skill 中的瓶颈优化建议，review ITERATIONS.md 寻找规律。制定计划后再继续。
- 连续 5 次迭代无改善：重新评估整体策略——查阅 Tiling 设计类 skill 重新设计切分方案，考虑是否需要换架构方向。
